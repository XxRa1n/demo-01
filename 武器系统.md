# 武器系统设计

> 本文件在原有「近战 / 远程 / 脱手 / 特殊」四类框架上，为每把武器补充**核心机制、关键参数（参考值）、升级路线**，并补全原本留空的「特殊武器」类，最后给出跨武器系统与落地架构建议。
> 标注「💡 建议新增」的条目为补充项，可按需取舍。
> 数值以现有手枪（Lv.1 ≈ 10 DPS）为基准校准，便于横向平衡。

---

## 设计原则

- **差异化优先**：每把武器有独特的「弹道 / 命中方式 / 节奏」，避免只是伤害高低的不同皮肤。
- **统一成长接口**：所有武器都走 `LEVEL_DATA` + `level_up()` + `is_max_level` + `get_level_display()`，无缝接入现有升级池。
- **复用全局属性**：伤害统一乘 `player.might`、冷却统一乘 `player.cooldown_mult`，被动升级对全武器生效。
- **DPS 横向平衡**：Lv.1 单体 DPS 控制在 8~14，差异体现在 AoE 占比、命中稳定性、爆发/持续、功能（击退/减速/燃烧）。

---

## 一、近战武器

共性：短距离、瞬时/周期 AoE、不依赖飞行体（或飞行体回旋），靠「范围 + 节奏」吃饭。

### 1. 投掷类（回旋刀 / 妈妈的刀）

- **定位**：围绕玩家旋转的持续切割武器，稳定近距 AoE。
- **机制**：发射刀刃实体，沿环形轨道绕玩家旋转（或飞出最大距离后原路返回），刀刃存在期间对接触敌人持续造成伤害（每敌人有 hit cooldown，防止单敌人无限受伤）。
- **关键参数**：旋转半径 ~90、刀刃飞行速度、每秒命中次数 ~3、单次伤害 ~4、存活时间（持续型可永久，往返型按周期）、击退。
- **升级路线**：+刀刃数量（多刃均布）→ +伤害 → +旋转半径/速度 → 命中附加流血 DoT → 满级「刀刃暴走」（半径随时间脉动）。
- **实现要点**：自定义 `OrbitBlade` 节点（沿用玩家坐标为圆心），Area2D + body 检测 + per-enemy cooldown 字典。

### 2. 挥砍类（剑 / 光刃）

- **定位**：朝面对方向挥出扇形斩击，瞬时 AoE + 强击退。
- **机制**：周期性在「移动方向 / 最近敌人方向」生成一道扇形斩击区域，瞬时命中范围内全部敌人，伤害 + 击退；斩击有短暂前摇动画。
- **关键参数**：扇形半径 ~120、扇形角度 ~100°、单次伤害 ~14、击退力度、挥砍间隔 ~0.9s、最大命中数（可设无上限）。
- **升级路线**：+斩击数量（同时多方向斩）→ +范围/角度 → +伤害 → +击退 → 满级「居合」（偶发大范围暴击斩）。
- **实现要点**：`SlashArea` Area2D，启用一帧检测扇形内敌人（距离 + 角度判定，或 collision polygon），瞬时 `take_damage` + 施加击退速度。

## 二、远程武器

共性：发射飞行体到 `Projectiles` 容器，差异在弹道、数量、命中特效。

### 3. 手枪（✅ 已实现 · 基准）

- 单发瞄准最近敌人、低伤高频、靠穿透与弹数成长。当前 `projectile_speed=700`、8 级成长（弹数 1→5、伤害 ×1.0→1.3、穿透 0→2）。作为其它武器的平衡锚点。

### 4. 霰弹枪

- **定位**：近距离爆发，贴脸伤害极高，远距衰减。
- **机制**：一次发射 6~8 颗弹丸，大角度扇形扩散，每颗独立伤害；弹丸飞行距离越远伤害越低（近全伤、远半伤），且有最大射程（超出消失）。
- **关键参数**：弹丸数 6、扩散角 ~50°、单发伤害 ~4（贴脸 6×4=24）、衰减曲线（0~250px 全伤，250~500 线性降至 50%）、射程 500、冷却 ~1.1s。
- **升级路线**：+弹丸数 → 二选一「聚焦（扩散↓集中↑）」/「散射（扩散↑弹数↑）」→ +伤害 → +射程 → 满级「独头弹」（每 3 发射一颗穿透大弹）。

### 5. 冲锋枪

- **定位**：极高射速、低单发、轻微散布、持续稳定 DPS。
- **机制**：极短冷却连发，每发带小随机散布角；单发伤害低但 DPS 稳定，适合风筝。
- **关键参数**：冷却 ~0.18s、随机散布 ±6°、单发伤害 ~3、弹数 1、射速受 `cooldown_mult` 强放大。
- **升级路线**：+射速（基础冷却↓）→ 双持（弹数 1→2 并行）→ +伤害 → -散布 → 满级「过载」（短暂爆发期射速翻倍）。

### 6. 火箭炮

- **定位**：慢速大弹、命中/到点爆炸 AoE、高伤低频。
- **机制**：发射慢速火箭，命中敌人或飞到最大距离时爆炸，对爆炸半径内全部敌人造成伤害 + 强击退。
- **关键参数**：爆炸半径 ~90、爆炸伤害 ~30、飞行速度 ~320、冷却 ~1.6s、击退强。
- **升级路线**：+爆炸半径 → +伤害 → +火箭数（多发分散）→ 穿透到指定点再爆 → 满级「集束」（爆炸再分裂 3 个小子弹）。

### 7. 手榴弹

- **定位**：抛物投掷、延迟引爆、定点清群。
- **机制**：投向目标点（最近敌人群中心），落地后短延迟爆炸 AoE；与火箭炮的区别是「延时 + 范围更大 + 可预判落点」。
- **关键参数**：投掷目标选取范围、引爆延迟 ~0.6s、爆炸半径 ~110、伤害 ~26、数量 1、冷却 ~1.4s。
- **升级路线**：+数量（一次多雷分散投）→ +范围 → +伤害 →「破片」（爆炸溅射小子弹）→ 满级「燃烧弹」（留下一片持续燃烧地面）。

### 8. 激光武器

- **定位**：持续光束、瞬时线性穿透、高频低单发、多目标。
- **机制**：锁定最近敌人，发射持续光束（射线），瞬间命中光束沿途所有敌人，按帧造成 DPS；光束有「持续 → 短歇」的节奏。
- **关键参数**：光束长度 ~520、宽度 ~14、DPS ~12（按帧分摊）、最大穿透目标数（可无限）、节奏（持续 1.2s / 间歇 0.3s）。
- **升级路线**：+宽度 → +DPS → +光束数（多条锁定不同目标）→ 命中减速 → 满级「聚能」（间歇期消失，持续光束 + 偶发灼烧）。
- **实现要点**：用 `RayCast2D` 或直接遍历沿线敌人（距离判负 + 投影），按 `delta` 累积伤害；渲染用 `Line2D`。

### 9. 声波武器

- **定位**：环形脉冲向外扩散，伤害 + 强推退，360° 无死角控场。
- **机制**：周期性从玩家发出一个环形声波，圆环不断向外扩散，命中敌人造成伤害 + 击退（环只判一次，靠扩散逐层打）。
- **关键参数**：扩散速度 ~280、最大半径 ~240、伤害 ~10、击退力度、波数 1、冷却 ~1.0s。
- **升级路线**：+波数（一次多环递推）→ +最大半径 → +伤害 → +击退 → 满级「共振」（命中敌人之间互相传导一次小伤害）。
- **实现要点**：`Shockwave` 节点带一个不断放大半径的 Area2D，记录已命中集合避免重复判定。

## 三、脱手武器

共性：召唤持续存在的独立实体（炮台 / 宠物），自主寻敌攻击；玩家专注走位。

### 10. 机器人炮台

- **定位**：定点/环绕炮台，自动开火，相当于第二个玩家在输出。
- **机制**：召唤炮台实体（固定位置，或环绕玩家公转），自动锁定射程内最近敌人射击，复用手枪类直射弹。
- **关键参数**：炮台数量 1、射程 ~360、射速 ~1.0s、单发伤害 ~8、持续时间（永久 / 可被击毁带血量）、环绕半径（环绕模式）。
- **升级路线**：+炮台数量 → +射速 → +伤害 → +射程 → 满级「导弹塔」（炮台改射爆炸弹）。

### 11. 宠物（猫 / 龙 等）

- **定位**：跟随玩家的自主攻击者，形态决定攻击方式。
- **机制**：宠物实体平滑跟随玩家，自动攻击范围内敌人——猫为快速近战咬，龙为中距吐息 AoE。
- **关键参数**：宠物数量 1、攻击范围 ~150、攻击间隔（猫 ~0.5s / 龙 ~1.0s）、伤害（猫 ~6 单体 / 龙 ~14 AoE）、移动跟随速度。
- **升级路线**：+宠物数量 → +伤害 → +攻速 → **进化**（猫→大猫、龙→巨龙，形态与攻击升级）→ 满级多形态并存。
- **实现要点**：宠物是带 `_physics_process` 的实体节点，加入 `Projectiles` 或新 `Minions` 容器。

## 四、特殊武器（原留空 · 💡 建议新增）

共性：机制独特，往往既是武器又是控场/功能。

### 12. 光环 / 领域

- **定位**：围绕玩家的持续伤害光环，贴身敌人持续掉血。
- **机制**：玩家身边一个圆形领域（可视为永久声波单元），范围内敌人按帧受 DPS + 轻微减速。
- **关键参数**：半径 ~110、DPS ~6、减速 ~20%。
- **升级路线**：+半径 → +DPS → +减速 → 多层异色光环（冰/火/毒叠加）→ 满级「黑洞光环」（向内轻微吸引）。

### 13. 闪电 / 连锁

- **定位**：攻击命中后伤害在敌人间跳跃，擅长密集群。
- **机制**：周期性对最近敌人放电，命中后伤害链式跳跃到附近 N 个敌人，每跳衰减。
- **关键参数**：跳跃次数 3、跳跃范围 ~160、首段伤害 ~10、每跳衰减 ×0.8、冷却 ~0.8s。
- **升级路线**：+跳跃次数 → +伤害 → +跳跃范围 → +首段目标数 → 满级「过载」（跳跃无视范围，跳全场）。

### 14. 黑洞 / 引力（💡）

- **定位**：聚怪 + 爆发清场。
- **机制**：在最近敌群处生成黑洞，持续吸引范围内敌人向中心移动，结束时爆炸造成 AoE 伤害。
- **关键参数**：吸引半径 ~180、吸引力度、持续时间 ~1.5s、结束爆炸伤害 ~40、冷却 ~3.0s。
- **升级路线**：+吸引范围 → +爆炸伤害 → +持续时间 → 双黑洞 → 满级「奇点」（吸引期间持续掉血）。

### 15. 时停 / 减速场（💡）

- **定位**：纯控场辅助武器，配合其它武器输出。
- **机制**：周期性展开减速场，场内敌人移速大幅降低（不直接造成伤害，靠全局武器收割）。
- **关键参数**：减速半径 ~220、减速幅度 ~60%、持续 ~1.5s、冷却 ~2.5s、自身 DPS 0（靠协同）。
- **升级路线**：+减速幅度 → +范围 → +持续时间 → 场内附加微量伤害 → 满级「绝对零度」（场内敌人额外承伤 +15%）。

## 五、跨武器系统

### 武器槽位与同时持有

- 玩家初始 1 个武器槽（手枪），通过升级可解锁到最多 **3~4 槽**。多武器并存时各自独立冷却、独立升级。

### 升级池与武器获取

- 升级面板从「被动 + 已持有武器升级 + **未持有武器·新武器选项**」池中抽 3。
- 新武器选项仅在有空闲槽时出现；满槽后只出已持武器的升级与被动。
- 武器升级条目沿用现有 `weapon_node.get_level_display()` 显示；满级后该武器退出升级池。

### 进化（满级合体，💡）

- 某武器满级 + 持有特定被动 / 另一把满级武器时，可触发「进化」合体为新形态（如 回旋刀 + 闪电 → 雷刃风暴）。作为后期目标，初期可不实现。

### 武器协同

- 异常状态统一化：燃烧 / 冰冻 / 中毒 / 流血作为共享 debuff，多武器可叠层触发，鼓励组合 build。

## 六、实现架构建议

### WeaponBase 基类

- 抽出 `weapon_base.gd`，承载通用：`weapon_level` / `is_max_level` / `LEVEL_DATA` / `level_up()` / `get_level_display()` / cooldown 计时（乘 `player.cooldown_mult`）/ 伤害（乘 `player.might`）。
- `weapon_pistol.gd` 改为继承 `WeaponBase`，只保留 `_fire()`；新武器同理，各武器的差异集中在 `_fire()` 与它使用的实体类。

### Projectile 变体

- `projectile_base.gd`（直线）保留为基类；新增子类：`explosive_projectile`（爆炸 AoE，给火箭/手雷）、`orbit_blade`（回旋，给投掷）、`shockwave`（扩散环，给声波）、`beam`（光束，给激光）。
- 统一加入 `Projectiles` 容器，统一受 `game_manager.game_over / is_paused` 控制。

### 近战 / AoE 检测

- 斩击、光环等不走 projectile：用一个瞬时启用的 `Area2D` + `get_overlapping_bodies()`，或 distance/angle 判定，避免和飞行体混管理。

### upgrade_manager 泛化

- 当前 `_get_weapon_node()` 硬编码 `"WeaponPistol"`，需改为遍历玩家下所有 `WeaponBase` 子节点：对每个未满级武器生成一条升级选项；有空闲槽时再追加「新武器」选项。
- 新武器获取建议用一个 `WEAPON_CATALOG`（场景路径 + 显示信息）登记表，`upgrade_manager` 据此实例化新武器节点挂到玩家。

### 优先实现顺序（建议）

1. **WeaponBase 重构**（让 pistol 继承）—— 所有后续武器前置依赖。
2. **upgrade_manager 多武器支持**—— 否则新武器无法获取/升级。
3. 第一把新武器选 **霰弹枪** 或 **回旋刀**（机制差异大、能验证 projectile 变体与 AoE 两条路线）。
4. 再铺开其余武器。
